01Introduction & Problématique

Ce projet s'inscrit dans le cadre du cours de Recherche Opérationnelle et porte sur l'optimisation des coûts de production dans une usine de fabrication de moteurs automobiles. L'objectif est de déterminer la combinaison de production qui maximise le profit global, tout en respectant les contraintes de stock et de capacité des machines.

L'usine produit trois types de moteurs — I4, V6 et V8 — chacun avec des coûts de fabrication et des prix de vente distincts. La modélisation repose sur la programmation linéaire (PL), résolue par la méthode du simplexe puis vérifiée via PuLP.

Ce type d'approche illustre comment la recherche opérationnelle permet à une entreprise de prendre des décisions de production éclairées, en exploitant au mieux ses ressources disponibles.

05Résolution — Simplexe & Python

Méthode du Simplexe

Le problème est mis sous forme canonique en ajoutant des variables d'écart (e₁–e₅) et d'excès (e₆–e₈) :

4x + 9y + 8z + e₁ = 5 000
2x + 3y + 4z + e₂ = 2 500
x + 3y + 4z + e₃ = 3 000
67x + 90y + 123z + e₄ = 25 000
55,5y + 27z + e₅ = 6 000
x − e₆ = 80  |  y − e₇ = 80  |  z − e₈ = 30
0 ≤ e₁, e₂, e₃, e₄, e₅, e₆, e₇, e₈

Résolution Python — PuLP

import pulp

problem = pulp.LpProblem("Maximize_W", pulp.LpMaximize)

x = pulp.LpVariable('x', lowBound=80, cat='Continuous')
y = pulp.LpVariable('y', lowBound=80, cat='Continuous')
z = pulp.LpVariable('z', lowBound=30, cat='Continuous')

problem += 1089 * x + 1809.5 * y + 3115 * z, "Objective"

problem += 4 * x + 9 * y + 8 * z <= 5000
problem += 2 * x + 3 * y + 4 * z <= 2500
problem += x + 3 * y + 4 * z <= 3000
problem += 67 * x + 90 * y + 123 * z <= 25000
problem += 55.5 * y + 27 * z <= 6000

problem.solve()

print("Status:", pulp.LpStatus[problem.status])
print("x =", int(pulp.value(x)))
print("y =", int(pulp.value(y)))
print("z =", int(pulp.value(z)))
print("Maximum W =", pulp.value(problem.objective))

Conclusion
